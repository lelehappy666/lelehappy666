# GitHub Overview 双列卡片布局设计

## 目标

优化 `README.md` 中“📊 GitHub Overview”区域的排列，使四张统计图在桌面端呈现为紧凑、整齐、间距统一的双列卡片布局，同时保持现有统计内容与东京夜主题不变。

## 布局方案

- 使用 GitHub Markdown 支持的 HTML `<table>` 构建稳定的 `2×2` 网格。
- 第一行展示 GitHub 综合统计与常用语言统计。
- 第二行展示连续贡献统计与活跃度趋势图。
- 每个单元格占据表格宽度的 50%，图片宽度设为 100%，由单元格统一控制尺寸与对齐。
- 表格整体居中，单元格顶端对齐，避免图片原始比例不同造成视觉错位。
- 移除原有多个独立居中容器和多余的 `<br/>`，减少纵向空白。

## 可访问性与兼容性

- 为四张统计图补充明确的中文 `alt` 文本。
- 不依赖自定义 CSS，避免使用 GitHub README 不支持的样式能力。
- 窄屏环境由 GitHub 页面自行缩放表格，保证内容不会脱离文档结构。

## 验收标准

- GitHub Overview 区域只保留一个居中的表格容器。
- 四张统计图按 `2×2` 顺序排列，双列宽度一致。
- 图片链接、用户名、主题和统计参数均保持不变。
- README 其余区域不发生改动。
